1 Prerequisites

2 Quickstart — eval harness

The single file to know is scripts/eval_baseline.py. It downloads real MLS data, builds features, trains Dixon-Coles and XGBoost, and evaluates them on held-out seasons using walk-forward validation. No database, no cron jobs, no setup beyond pip install.

source venv/bin/activate
python scripts/eval_baseline.pybash

What the eval structure means

Walk-forward 3-way split (no future leakage)
├── train  = all seasons < cal_season   model fitting
├── cal   = cal_season = test_season − 1   temperature calibration + meta-learner
└── test = test_season   final held-out evaluation — never touched until scoring

Test seasons are 2023 and 2024 (2022 is skipped because 2021 — the cal fold — was a COVID season). Training data starts from 2017; 2020 and 2021 are excluded entirely as COVID anomalies.

3 Configuration knobs in eval_baseline.py

All tunable constants are at the top of the file (~lines 52–70). Edit these and re-run to test the effect.

4 Adding or removing features

Current base feature set

The Base set is assembled around line 750 from named lists:

# scripts/eval_baseline.py — feature set assembly

_BASE_ELO  = ["elo_diff", "home_elo", "away_elo"]

_BASE_XG   = (
    [f"{r}_xg_roll_{w}"  for r in ("home", "away") for w in XG_WINDOWS]
    + [f"{r}_xga_roll_{w}" for r in ("home", "away") for w in XG_WINDOWS]

# GK quality — promoted from A/B (Δ=+0.0034). Excluded if ASA data unavailable

_FEAT_BASE = _BASE_ELO + _BASE_XG + _BASE_FORM + _BASE_GK + ["is_playoff"]python

How to add a new feature

  • 1

    Compute it in add_rolling_features() (~line 325). This function iterates matches in date order and tracks per-team histories. Add your calculation here so it respects temporal order (no future leakage).

    # Example: rolling draw rate (last N matches)
for fw in (10,):
    draws_hist = [1 if p == 1 else 0 for p in pts_hist[-fw:]]
    res[f"{role}_draw_rate_{fw}"].append(
        np.mean(draws_hist) if draws_hist else 0.27
    )python
  • 2

    Declare it in res at the top of add_rolling_features():

    res["home_draw_rate_10"] = []
res["away_draw_rate_10"] = []python
  • 3

    Add it to the AB_SETS dict (~line 800) as a new test group:

    AB_SETS["+DrawRate"] = _FEAT_BASE + ["home_draw_rate_10", "away_draw_rate_10"]python
  • 4

    Run the eval and read the A/B table. If Δ > 0.001 averaged across test seasons, the feature passes and can be promoted into _FEAT_BASE.

Leakage check: never use data from match day N+1 to predict match N. The rolling feature loop processes rows in date order; always append to history after reading it. The loop structure already does this — just follow the same pattern.

5 Running an A/B test

The eval script runs all entries in AB_SETS automatically and prints a comparison table at the end. The decision rule is:

Brier delta (Base − candidate)Decision
> 0.001✅ KEEP — promote to Base
0 to 0.001~ marginal — monitor across more seasons
< 0❌ DROP — hurts performance

The script also picks the best feature set by calibration-set Brier to use in the stacking ensemble. This avoids test-set leakage when selecting which features the meta-learner sees.

# Example: test a new feature group called "+DrawRate"

# 1. Compute draw rate in add_rolling_features() (see §4)

# 2. Add to AB_SETS dict (~line 800):
AB_SETS["+DrawRate"] = _FEAT_BASE + ["home_draw_rate_10", "away_draw_rate_10"]

# 3. Run and read the table:
# A/B feature test (XGBoost Brier, avg over 2 seasons):
#   Feature set     Brier   Δ vs Base   Keep?
#   Base           0.6381           —       —
#   +DrawRate      0.6360     +0.0021     YES  ← passes thresholdpython
Currently only 2 test seasons are available (2023, 2024 — 2022 cal fold was COVID). A Δ of ~0.001 on 2 seasons has a wide confidence interval. Be conservative; a feature that passes by Δ=0.0011 may be noise. Aim for Δ ≥ 0.002 before promoting.

Trying a different XGBoost objective

The script currently uses multi:softprob (3-class probabilities). You could experiment with a custom loss by subclassing xgb.XGBClassifier and providing obj + feval parameters, but the standard objective is hard to beat on well-calibrated outputs.

Dixon-Coles decay half-life

Change DC_DECAY_HL at the top of the file. A shorter half-life (e.g., 60 days) makes the model react faster to recent form; longer (180 days) makes it more stable but slower to update after roster changes.

The ensemble blend (capped-DC convex blend)

The final model is not a plain average or a logistic-regression meta-learner anymore. It's a scalar convex blend w·XGB + (1−w)·DC, where w is fitted on the calibration fold by minimising Brier, constrained so Dixon-Coles contributes at most 30% (w ≥ 0.70). This was adopted on 2026-05-30 (arch-capped-dc) because an unconstrained meta-learner let DC blow up the 2024 season (Brier 0.6523) — capping it fixes 2024 while keeping DC's 2022–23 synergy.

Look for the block that fits _blend_brier / minimize(...) in the meta-learner section (search def _blend_brier). To experiment with the cap, change the lower bound on w (e.g. bounds=[(0.60, 1.0)] = 40% DC cap, (1.0, 1.0) = pure XGB). Note from the 2026-06-06 loop: the cap is insensitive between 20–30% on the final output (Δ ≈ +0.0001), so don't expect Brier movement from small cap changes — the bigger lever is the features and the raw models, not the blend weight.

Watch the metric. When you tweak the blend, judge it by the Ensemble stacked row and the per-season ens_stacked_brier table at the bottom of the run — those are the calibrated final outputs. Don't judge by the raw blended probabilities before calibration; they can look better while the reported number doesn't move.

Pattern: two confirmed A/B KEEPs (+TZ_Pythag and +VenueGoalDiff, both Δ=+0.0013) but neither beats +All in the per-fold cal-fold BestAB contest. The +All set, despite including many DROP features, wins because XGBoost's built-in feature selection (colsample, tree structure) handles the noise internally. Combining KEEPs into new sets (+MargCore, +MargCoreVG) causes interference rather than stacking.

The 2024 robustness gate: any feature added to _ALL_EXTRA (and therefore into +All) risks regressing 2024 if XGBoost uses those extra features in the 2024 BestAB fold. +VenueGoalDiff added to +All caused 0.0005 regression in 2024. Keep new features as standalone AB sets until proven to not harm 2024 when included in +All.

Calibration: Temperature Scaling

The script uses temperature scaling — a single scalar T that sharpens or softens all probabilities. T > 1 makes predictions more uncertain; T < 1 makes them more confident. The optimal T is found by minimising negative log-likelihood on the cal set.

# calibrate_multiclass() — around line 899
def _nll(T: float) -> float:
    log_p = np.log(np.clip(raw_cal, 1e-9, 1.0)) / max(T, 0.1)
    ...
T = minimize_scalar(_nll, bounds=(0.3, 5.0), method="bounded").x

# To see what T the model is choosing, print it:
print(f"Temperature T = {T:.3f}")
# T > 1 = model is over-confident; T < 1 = model is under-confidentpython

9 Reading the output

Brier score

The primary metric. For 3-class predictions (home/draw/away), Brier score = mean squared error across all probability assignments. Range 0–2; lower is better.

Feature importances

Reported as gain — the average improvement in the loss function from splits on that feature. Roughly equal gain across features (currently ~4% each) suggests the model has many weak signals rather than a few strong ones. A dominant feature (≥10% share) would indicate high-value signal.

Calibration error

The max decile error measures the worst-case gap between predicted probability and actual frequency in any probability bucket. Target is <0.05. Current value is ~0.18 — meaning a 60% prediction might actually correspond to a 42%–78% true frequency. This is too wide for Kelly sizing.

Per-class Brier

Draws (class 1) are hardest to predict — structurally, teams that draw tend to continue drawing, but predicting which specific match ends in a draw is very hard. Home wins are easier (home advantage is real and consistent).

A/B table delta sign convention

Positive Δ = improvement (Base Brier − candidate Brier; lower is better, so positive means candidate is lower). A negative Δ means the feature increased the Brier score, i.e., made things worse.

10 Troubleshooting

SymptomCauseFix
SSL: CERTIFICATE_VERIFY_FAILED System clock is ahead of cert validity window The script patches asa.session.verify = False automatically. If you see this, ensure you're running the latest version of the script.
DC fit hangs >5 min L-BFGS-B not converging on large data Reduce recent_seasons in fit_dc() from 4 to 3, or reduce maxiter in the minimize call.
Season 2022 skipped Cal fold (2021) is a COVID season — excluded from training data Expected behaviour. Results are averaged over 2023–2024 only.
get_game_xpass not found ASA API doesn't expose per-game PPDA Expected — PPDA and possession are skipped automatically. Season-level PPDA via get_team_xpass is used instead.
XGBoost A/B trains very slowly Too many feature sets × hyperparameter combinations Reduce AB_SETS to just {"Base": ..., "+YourFeature": ...} for focused testing. Also reduce the inner grid to fewer combos.
GK features show 0% coverage ASA didn't return goalkeeper data for early seasons Normal for seasons before ~2017. The feature uses a 1–2 season lag fallback, so coverage is <100% for early test folds.
ImportError: No module named 'rpy2' R integration package not installed This is intentional — rpy2 is commented out in requirements.txt. The Bayesian model is disabled by default. Uncomment rpy2 and install R if you need it.
